# 04 路由与链接问题排查

## 1. 典型现象

- 页面上 `Posts` / `Tags` / `Categories` / `Read More` 点击“无反应”
- URL 变化了，但页面内容看起来不变

## 2. 这类问题的常见根因

### 根因 A：Nginx `try_files` 未匹配目录

如果使用：

```nginx
try_files $uri /index.html =404;
```

则 `/posts/` 这类目录请求可能直接回退到首页 `index.html`，表现为“点了没跳”。

应改为：

```nginx
try_files $uri $uri/ /index.html =404;
```

本项目 `nginx/default.conf` 已按此修复。

### 根因 B：配置改了但容器未重载

修改 `nginx/default.conf` 后需重启容器，否则仍使用旧配置。

### 根因 C：Hugo 未生成对应页面

若 `/posts/index.html`、`/tags/index.html` 等不存在，点击也会异常回退或 404。

### 根因 D：博客静态资源被错误回退到首页

如果博客站点使用：

```nginx
try_files $uri $uri/ /index.html =404;
```

则像 `/favicon-32x32.png`、`/favicon.ico`、`/site.webmanifest` 这类根路径静态资源在未命中时，可能被错误回退为首页 HTML，表现为：

- 浏览器请求图标但拿到的是 `200 text/html`
- 本地源站正常，经过 CDN/Cloudflare 后持续命中错误缓存

博客站点应改为：

```nginx
try_files $uri $uri/ =404;
```

这样静态资源缺失时会明确返回 `404`，避免将首页 HTML 缓存成资源文件。

## 3. 标准排查步骤

1. 确认产物存在：
   - 检查 `hugo_public` 中是否有 `/posts/index.html`、`/tags/index.html`、`/categories/index.html`
2. 确认首页链接：
   - 查看首页 HTML 中 menu 与 `Read More` 的 `href`
3. 对比不同 URL 内容：
   - 对 `/`、`/posts/`、`/tags/`、`/categories/`、具体文章页抓取 HTML 并比较摘要/校验值
4. 若配置已更新但行为未变：
   - 重启 `nginx`
5. 若经由 Cloudflare 等 CDN 访问仍异常：
   - 对异常 URL 做单文件缓存清理，再复测

## 4. 与 `HUGO_BASE_URL` / `baseURL` 的关系

- `HUGO_BASE_URL`（构建时传给 Hugo 的 `--baseURL`）主要影响 canonical、RSS、社交元信息，不是“点击无反应”的主因。
- 但上线前应改为实际域名，避免 SEO 和分享链接异常。若未设置，默认使用 `http://127.0.0.1:<HOST_PORT>/`。

## 5. 快速验证口径

- `GET /` 与 `GET /posts/` 的 canonical 应不同
- `GET /posts/test/` 应返回文章页 canonical，而不是首页 canonical
- 菜单 `href` 应对应真实存在的目录页
- `GET /favicon-32x32.png` 应返回 `200 image/png`，而不是 `200 text/html`
