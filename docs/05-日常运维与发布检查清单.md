# 05 日常运维与发布检查清单

## 1. 启动前检查

- Docker 与 `docker compose` 可用
- `MARKDOWN_DIR` 存在且无空格路径
- 端口未被占用，且 `HOST_PORT != EDITOR_PORT`
- 若启用 adapter，脚本可执行
- 若使用默认 watcher，宿主机为 Linux（amd64/arm64）

## 2. 启动后检查

- `http://127.0.0.1:<HOST_PORT>/` 可访问博客首页
- `http://127.0.0.1:<EDITOR_PORT>/` 可访问编辑器
- `http://127.0.0.1:<HOST_PORT>/waline/ui` 可打开评论管理后台
- Waline 登录/评论接口无协议错误、无 401/500 异常
- 菜单和文章详情可正常跳转
- `/_assets/...` 资源可访问
- `/favicon-32x32.png`、`/favicon.ico`、`/site.webmanifest` 等根路径静态资源可访问，且 `Content-Type` 正确
- watcher 状态正常（查看 `.markwatch.log`）

## 3. 内容更新检查

- 触发 `markcompose.sh build` 后，页面能看到新内容
- 文章 front matter 字段完整（`title/date/draft/slug`）
- `*.md` 互链跳转正常（`relref` 生效）
- 图片路径在浏览器可访问（尤其是 `_assets`）

## 4. 故障定位优先级

1. 看 `.markwatch.log`（watch 是否触发构建）
2. 手动执行 `./markcompose.sh build .env.runtime`（验证构建链路）
3. 检查 nginx 路由规则（博客应使用 `try_files $uri $uri/ =404`，避免把静态资源错误回退为首页）
4. 检查 `hugo_public` 实际产物是否包含目标页面

## 5. 停止与清理

- 常规停止：`./markcompose.sh stop`
- 连卷清理：`./markcompose.sh stop -v`
- `-v` 会删除 `waline_data`，Waline 用户/评论会一起丢失
- 需要“干净重建”时再清卷，避免误删已发布产物

## 6. 上线前建议

- 在 `.env` 中将 `HUGO_BASE_URL` 改为实际线上域名
- 复测关键入口：首页、列表页、详情页、标签页、分类页、静态资源
- 若站点走 Cloudflare/CDN，修改 favicon、manifest、首页元信息后，对对应 URL 执行单文件缓存清理
- 固化一份最小回归清单（菜单、Read More、资源加载、RSS）
